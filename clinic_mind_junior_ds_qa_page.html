3. What steps do you follow to ensure data quality, accuracy, and consistency?

▶
I follow a structured process:
  1. Source & schema checks: Confirm table definitions, data types, and primary/foreign keys; verify that I understand how the data is generated.
  2. Sanity checks: Use SQL and Python (pandas) to check row counts, distinct keys, null distributions, and basic statistics to spot obvious anomalies.
  3. Business-rule validation: Verify that values make sense in context (for example, no negative counts where impossible, timestamps in valid ranges).
  4. Outlier and duplication checks: Identify outliers and duplicate records, and either correct, cap, or exclude them based on agreed rules.
  5. Reproducible cleaning pipeline: Implement the cleaning logic in reusable code (SQL scripts or Python functions) so the same steps run consistently for new data.
  6. Spot checks and peer review: Compare sample records before and after processing and, when possible, review assumptions with a domain expert or teammate.
This combination of technical checks and business validation helps keep the data consistent and trustworthy for analysis.
Sanity checks Reproducible pipelines Business-rule validation

4. Have you worked with large datasets? If yes, describe the challenges you faced and how you handled them.

▶
Yes, I have worked with large datasets both in industry and in academic projects. The main challenges were memory limits, slower query times, and ensuring that intermediate steps remained efficient.

To handle this, I typically:
  • Push as much filtering and aggregation as possible into SQL so that Python only receives the reduced, relevant data.
  • Work with sampled subsets for rapid prototyping, then scale to full data once the logic is validated.
  • Use chunked processing and vectorized operations in pandas instead of row-by-row loops.
  • Cache intermediate results (for example, summary tables) to avoid recomputing heavy transformations repeatedly.
These approaches have helped keep workflows reliable and performant even when the raw data volume is large.
Sampling & chunking Push work to SQL Vectorized operations
